Android 智能手表测试

Posted on 2019-01-28

背景

  • 测试设备：HUAWEI WATCH 2 Pro
  • Wear APP != Phone APP
  • 手表可以自行连接wifi上网，也可以蓝牙连接手机，通过手机上网

关于 Wear APP

  • 使用Android Studio只允许在手机上安装Phone APP，在手表上安装Wear APP
  • 使用adb install可以在手表上安装Phone APP，支持插桩，正常运行，生成测试报告
  • 截图和dom dump速度较慢（一般dump需要5s）

Wear 各指标及测量方法

  • 电池信息：电量、温度
    • 拨号盘输入*#*#4636#*#*
  • 电量/power consumption：
    • Google提供的开源电池历史数据分析工具battery-historian
    • Android自带adb shell dumpsys batterystats可以生成报告
    • 第三方硬件测量：High Voltage power monitor
  • 温度：
    • 电池温度
    • CPU温度
    • 第三方软件获取：360，金山…
    • 通过读取手机传感器sensor的温度近似于手机CPU温度
    • 通过读取CPU信息来获取
  • Memory
    • adb shell dumpsys meminfo <>
    • logcat
    • Android Profiler
  • Network
    • Android Profiler
  • CPU
    • Android Profiler
  • Latency
    • Launch time
    • update time
    • response time
    • ntpd可以同步多设备时间
  • GPU/FPS
    • adb shell dumpsys gfxinfo…
    • systrace
    • logcat
  • 网络优化
    • AS自带Network Monitor: 网络请求数量及访问速率
    • 抓包工具
    • 网速也可以通过开源库编码获得
  • 流量
    • TrafficStats类
    • tcpdump抓包，wireshark可以读取
    • 综合使用adb shell cat /proc/uid_stat/uid/tcp_rcv和tcp_snd命令
  • loss (ping)
    • *#*#4636#*#*
  • Android性能测试的指标：
    • 启动时间
    • 内存
    • CPU
    • GPU
    • 功耗
    • 流量
  • 其它
    • 渲染性能
    • disk access
    • CPU availability
    • QoS
    • round trip latency

定位分类

  • 健康
